How Does It Work?
01 / PIPELINE
Data Simulation & Feature Engineering
Synthetic logs are generated for user logins, file access, USB usage, and emails, mimicking real organisational activity. Features extracted include login/logout patterns, per-day activity rates, out-of-session file access, graph centrality metrics, and NLP features from email subjects (keyword flags, sentiment, length).
02 / MODELS
Anomaly Detection Algorithms
Isolation Forest — randomly partitions data; anomalies are isolated faster (shorter average path length in trees).

One-Class SVM — finds a boundary in feature space enclosing normal data; points outside are flagged as anomalies via RBF kernel methods.

Autoencoder (MLP) — learns to compress and reconstruct normal behaviour; high reconstruction error indicates an anomaly.
03 / GRAPH
Graph Analysis
Users, files, and devices are represented as nodes; access and usage events form edges. Degree centrality measures activity level; betweenness centrality identifies potential information brokers. The at-risk subgraph focuses on high-risk users and their direct connections.
04 / XAI
Explainability
SHAP — based on cooperative game theory; attributes model predictions to each feature by averaging over all possible feature orderings.

LIME — perturbs input data locally and fits a simple interpretable model to approximate complex model behaviour.
05 / SIMULATION
Red Team Simulation
Injects malicious behaviours — after-hours access, mass file downloads, suspicious USB usage — into the dataset to test and benchmark the detection pipeline's sensitivity and precision.
