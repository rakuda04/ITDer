How Does It Work?
01 / PIPELINE
Data Simulation & Feature Engineering
Synthetic logs for logins, file access, USB, and emails. Features include login/logout patterns, per-day activity rates, out-of-session file access, graph centrality, and NLP features from email subjects.
02 / MODELS
Anomaly Detection Algorithms
Isolation Forest — anomalies isolated faster via shorter path lengths.

One-Class SVM — kernel boundary around normal data; outliers flagged.

Autoencoder — high reconstruction error = anomalous behaviour.
03 / GRAPH
Graph Analysis
Users, files, and devices as nodes. Degree and betweenness centrality expose information brokers and lateral movement.
04 / XAI
Explainability
SHAP — game-theory attribution per feature.

LIME — local perturbation approximates complex decisions.
05 / RED TEAM
Red Team Simulation
Injects after-hours access, mass downloads, suspicious USB usage to benchmark detection pipeline sensitivity and precision.
06 / INDICATORS
What Does "RED" Mean in the Table?
The RED badge displayed next to a username means that user is a confirmed Red Team member — a simulated insider threat injected into the dataset to benchmark the detection system.

Red Team users intentionally perform malicious behaviours such as:

  • Accessing files outside of normal working hours
  • Mass downloading of sensitive documents
  • Unusual or excessive USB device usage
  • Sending emails with suspicious keywords

These users should always appear at or near the top of the anomaly ranking. If they do not, it indicates the model needs retraining or feature tuning. The RED badge is ground-truth data — in a real deployment this would represent a confirmed threat actor identified through investigation.
